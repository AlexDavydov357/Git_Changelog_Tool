# 📝 Git Changelog Tool 🔥️

[![Python](https://img.shields.io/badge/Python-3.8+-blue?logo=python)](https://www.python.org/)
[![wxPython](https://img.shields.io/badge/wxPython-4.2+-green?logo=python)](https://wxpython.org/)
[![License](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

> **Удобный GUI-инструмент для ведения журнала изменений и управления Git-коммитами**  
> Автоматически форматирует историю изменений, поддерживает темную тему, работу с несколькими репозиториями и тестовые коммиты.

---

## 🌟 Особенности

| ✅ Функция | Описание |
|-----------|----------|
| 📅 **Динамическая дата** | Автоматическое добавление текущей даты (или выбор из календаря) |
| 📝 **Построчное редактирование** | Добавление/редактирование/удаление изменений с поддержкой чекбоксов («закоммичено?») |
| 💾 **Сохранение в JSON** | `changelog.json` — структурированное хранение с историей и флагами |
| 📄 **Экспорт в Markdown** | Автоматическая генерация `CHANGES.md` по шаблону |
| 🧪 **Тестовые коммиты** | Кнопка `🚀 Test` создаёт и пушит тестовый коммит в `origin` и `backup` |
| ↩️ **Откат тестовых коммитов** | `↩️ Revert` — откатывает последний коммит и пушит откат |
| 🌓 **Темная/светлая тема** | Переключение в реальном времени (кнопка-переключатель в шапке) |
| 👁️ **Последний коммит** | Быстрый просмотр последнего коммита без терминала |
| 🚀 **Push в несколько репозиториев** | Поддержка `origin`, `backup`, и `push-all` |

# 🎮 Как пользоваться
## 1. Добавить изменение
### Выберите дату (или оставьте текущую)
### Введите описание в поле ввода → нажмите Enter или кнопку 📝 Добавить
### ✅ Чекбокс автоматически ставится (означает: «ещё не закоммичено»)
## 2. Редактировать/удалять
### Кликните по строке → редактируйте текст (или нажмите F2)
### Удалите: Delete / Backspace или кнопка 🗑️ Удалить
## 3. Закоммитить изменения
### Отметьте нужные строки чекбоксами (✅ = «закоммичено»)
### Нажмите 💾 Commit → появится предпросмотр сообщения
### Подтвердите → коммит создастся, а флаги обновятся
## 4. Отправить в репозиторий
### ⬆️ Push Origin — пуш в origin
### ⬆️ Push BackUP — пуш в backup
### 🔄 Push All — пуш в оба сразу
## 5. Тестовые операции
### 🚀 Test — создаёт test_commit.txt, коммитит и пушит
### ↩️ Revert — откатывает последний коммит и пушит откат
## 6. Экспорт
### 📄 Export to CHANGES.md — генерирует Markdown-файл по шаблону:
